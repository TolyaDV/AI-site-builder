## 1. SaaS / Product landing

**Типова ціль**: leads / trials / signups.

**Landing-структура**:
1. **Hero** — problem-first заголовок + product name + primary CTA («Спробувати безкоштовно» / «Отримати демо»).
2. **Social proof smal** — логотипи 4-6 клієнтів або метрика («10 000+ команд використовують»).
3. **Problem / Pain** — що ламається без продукту (коротко, 2-3 проблеми з іконками).
4. **Solution / How it works** — 3 кроки або візуалізація flow.
5. **Features** — 3-6 ключових фіч з іконками + 1 речення кожна.
6. **Integrations** (якщо релевантно) — логотипи інтегрованих сервісів.
7. **Testimonials** — 2-3 розгорнуті цитати з фото і роллю.
8. **Pricing** — 2-3 тарифи або CTA «Дізнатися ціну».
9. **FAQ** — 5-7 запитань, які реально ставлять.
10. **Final CTA** — великий заклик + secondary «Забронювати демо».

**Чому такий порядок**: attention → довіра → біль → розв'язання → деталі → соц. докази → умови → зняття заперечень → дія.

**Multipage-варіант**: Home / Features / Pricing / Customers / Docs / Blog / Contact. Використовуй якщо продукт складний і має окремі buyer-persona flows.

**Коли відходити**: якщо продукт self-serve і дешевий → вкорочуй до 5-6 секцій, прибирай FAQ+Testimonials. Якщо enterprise B2B → додай Security & Compliance окремим блоком перед Pricing.

## 2. E-commerce (магазин)

**Типова ціль**: sales.

**Header (sticky)**: лого, пошук, категорії, кошик, акаунт.

**Головна сторінка (Landing-стиль навіть для multipage)**:
1. **Hero banner** — поточна промо-кампанія або brand statement з CTA «В каталог» / «Купити».
2. **Категорії** — 4-8 плиток з іконками/фото.
3. **Featured / Bestsellers** — 4-8 товарів у сітці.
4. **New arrivals** — 4-6 товарів.
5. **USPs strip** — безкоштовна доставка / повернення / гарантія / підтримка (іконки + 1 рядок).
6. **Testimonials / Instagram feed** — соц. докази.
7. **Newsletter** — знижка за підписку.

**Multipage (обов'язково для e-commerce)**:
- `/` (головна)
- `/shop` або `/catalog` (список з фільтрами)
- `/product/[slug]` (картка товару)
- `/cart`
- `/checkout`
- `/account` (якщо є)
- `/about`, `/contact`, `/shipping`, `/returns` (статичні)

**Чому такий порядок**: зачепити (банер) → навігація (категорії) → тригери купівлі (bestsellers) → зняти страхи (USPs) → утримати (newsletter).

**Коли відходити**: luxury-бренди часто прибирають promo-банер, ставлять editorial hero; niche-магазини одного товару роблять pure landing без каталогу.

## 3. Локальний сервіс

Клініка, салон, ресторан, автосервіс, фітнес-студія.

**Типова ціль**: візит / бронювання / дзвінок.

**Landing-структура** (підходить 80% кейсів):
1. **Hero** — що робимо + де + CTA «Записатись» / «Забронювати столик». Часто фото закладу як фон.
2. **Послуги / Меню** — список з цінами або діапазоном.
3. **Про заклад / команду** — 2-4 абзаци + фото, регалії, роки досвіду.
4. **Галерея** — 6-12 фото закладу/робіт/страв.
5. **Відгуки** — Google rating + 3-5 цитат. Google-integration або скріншоти.
6. **Локація і години** — карта + години + телефон.
7. **Booking-форма / Контакти** — форма або номер.
8. **FAQ** (опційно) — для специфічних кейсів (підготовка до процедури, умови).

**Multipage-варіант** (коли послуг багато і кожна потребує свого пояснення):
- `/` (головна з коротким оглядом)
- `/services/[slug]` (кожна послуга окремо)
- `/team` (якщо команда — важливий фактор вибору)
- `/gallery`
- `/contact` (з мапою і формою)

**Чому такий порядок**: що+де → що пропонуєте → хто ви → докази якості (фото) → соц. докази (відгуки) → практика (як дістатись/записатись).

**Коли відходити**: premium-сегмент (luxury-спа, мішленівський ресторан) → дуже editorial стиль, прибирай явний pricing, roble акцент на атмосфері. Екстрена послуга (автосервіс-евакуатор) → телефон у Hero, секція «де ми зараз» з картою.

## 4. Agency / Studio / Consulting

Дизайн-студія, маркетинг-агентство, веб-розробка, консалтинг.

**Типова ціль**: leads (запит на проєкт).

**Landing-структура**:
1. **Hero** — що ми робимо + кому + CTA «Розпочати проєкт» / «Зв'язатися».
2. **Featured work / Case studies** — 3-6 робіт з картинкою + 1 рядок опису. Клік → кейс-стаді.
3. **Services** — 3-5 напрямків + 1-2 речення кожен.
4. **Process** — 3-5 кроків співпраці.
5. **Team / About** — коротко про компанію, фото команди.
6. **Clients / Logos** — 8-15 логотипів клієнтів.
7. **Testimonials** — 2-3 розгорнуті відгуки з фото клієнта.
8. **Final CTA** — форма заявки або calendar booking.

**Multipage-варіант**:
- `/` (landing-стиль)
- `/work` (grid усіх проєктів)
- `/work/[slug]` (кейс-стаді)
- `/services`
- `/services/[slug]` (опційно — якщо кожен сервіс вимагає глибокого опису)
- `/about`
- `/blog` (опційно — для SEO)
- `/contact`

**Чому такий порядок**: хто ми+що робимо → докази якості (кейси одразу!) → що пропонуємо → як працюємо → хто в команді → довіра (клієнти+відгуки) → дія.

**Коли відходити**: studio-першого-класу часто робить editorial-stiль з величезною галереєю робіт і майже без тексту; consulting-агенція — мінімум картинок, багато expertise-контенту (white papers, speaking engagements).

## 5. Portfolio

Дизайнер, фотограф, ілюстратор, режисер.

**Типова ціль**: image + leads.

**Landing-структура** (мінімалістична):
1. **Hero / Intro** — ім'я + роль + 1 речення позиціонування + CTA «See work» / «Contact».
2. **Featured works** — grid 6-12 робіт. Клік → кейс або lightbox.
3. **About** — коротка біо, approach, clients/publications, regalia.
4. **Clients / Publications** (логотипи) — якщо є.
5. **Contact** — форма + email + соцмережі.

**Multipage-варіант**:
- `/` (hero + обране)
- `/work` (грід всіх робіт)
- `/work/[slug]` (кейс з процесом і фінальним результатом)
- `/about`
- `/contact`

**Чому такий порядок**: краще менше тексту — роботи говорять самі. Текст тільки щоб відповісти на «хто ти» і «як зв'язатись».

**Коли відходити**: комерційний портфоліо для B2B-клієнтів — додавай services + rates + process. Чисто-artsy портфоліо — прибирай все крім hero + work + contact.

## 6. Курс / Освітній продукт

Онлайн-курс, навчальна програма, майстер-клас.

**Типова ціль**: enrollments / leads.

**Landing-структура**:
1. **Hero** — promise («Навчись X за Y тижнів») + primary CTA «Записатись» або «Дізнатись більше».
2. **Problem / Agitate** — з якою болючою ситуацією живе учень зараз.
3. **For whom / Not for whom** — хто отримає максимум, хто ні (парадоксально підвищує довіру).
4. **Outcomes** — що учень вмітиме після (список 4-6 пунктів).
5. **Curriculum / Program** — модулі або тижні з коротким описом.
6. **Instructor bio** — хто викладач, досвід, чому йому можна довіряти.
7. **Testimonials / Graduates** — відгуки учнів з фото, бажано з результатами (до/після).
8. **Pricing / Plans** — вартість, що входить, формат оплати.
9. **FAQ** — 6-10 запитань (формат, підтримка, доступ, повернення).
10. **Final CTA** — великий блок «Готовий почати?» + форма.

**Чому такий порядок**: обіцянка → біль → self-identification → конкретна користь → як саме → довіра до людини → довіра через соц. докази → умови → зняття заперечень → дія.

**Коли відходити**: free-курс-лід-магніт → вкороти до hero + outcomes + signup. Дорогий enterprise-training → додай B2B-блок (case studies компаній, корпоративні пакети) перед Pricing.

## 7. Подія / Конференція

Конференція, митап, фестиваль, турнір.

**Типова ціль**: продаж квитків / реєстрація.

**Landing-структура**:
1. **Hero** — назва + дата + місто + primary CTA «Купити квиток» / «Зареєструватись». Часто countdown-таймер.
2. **About / What** — що це за подія, для кого, у форматі 2-3 абзаци.
3. **Speakers** — фото + ім'я + роль + тема. Grid 6-20 спікерів.
4. **Schedule / Program** — розклад по днях або потоках.
5. **Venue** — де, як дістатись, мапа.
6. **Tickets / Pricing** — 2-3 типи квитків з differences.
7. **Sponsors / Partners** — логотипи партнерів.
8. **Past editions** (якщо не перше) — фото/відео попередніх років.
9. **FAQ** — refund-policy, dress code, трансляція, їжа.
10. **Final CTA** — великий блок з даними і кнопкою.

**Чому такий порядок**: що+коли+де+кнопка одразу → деталі для сумнівних → хто виступає (головний магніт) → коли саме що → куди прийти → скільки → довіра через партнерів → соц. докази минулих → зняти заперечення → дія.

**Коли відходити**: закрита VIP-подія → прибирай pricing, ставь apply-only форму; безкоштовний митап → вкоротити до hero + speakers + venue + register.

## 8. Блог / Контент-сайт

Персональний блог, newsletter, контент-медіа.

**Типова ціль**: audience / subscribers.

**Головна (landing-стиль)**:
1. **Hero / Featured post** — найсвіжіший або найважливіший пост великим банером.
2. **Featured posts** — 3-6 топ-постів у grid.
3. **Categories / Topics** — 4-8 тематичних плиток.
4. **Newsletter signup** — форма + причина підписатись.
5. **About the author** — 2-3 абзаци + фото.
6. **Archive / Recent** — список останніх 10 постів.

**Multipage (обов'язково)**:
- `/` (головна)
- `/blog` (весь архів з фільтрами)
- `/blog/[slug]` (пост)
- `/categories/[slug]` або `/tags/[slug]`
- `/about`
- `/newsletter` (окрема landing для підписки)
- `/contact`

**Чому такий порядок**: дати прочитати сильний контент одразу (hero) → показати що є ще → навігація за темами → моментальний CTA (newsletter) → хто автор → залучити глибше.

**Коли відходити**: nielsdeveloper-блог → вкорочуй до hero + archive + about; paid-newsletter з платними рівнями → виноси pricing на головну.

## Як використовувати це скіл

1. Знайди ніша-ціль найближчу до твоєї.
2. Перевір секцію «Коли відходити» — можливо твій кейс тут.
3. Якщо знайшов — адаптуй (прибери зайве, додай специфіку), не копіюй мертвою буквою.
4. Якщо немає — переходь у principles-mode (SKILL.md Крок 3).
5. Завжди перевіряй через `cross-niche-patterns.md` чи не треба додати diff-режим.
